- 2026-05-01: detached checkout에서는 `git rev-parse --abbrev-ref HEAD`가 `HEAD`를 반환하므로 현재 구현은 `ai-workflow/memory/HEAD`를 branch memory로 사용할 수 있다. 영향 범위는 상태 생성뿐 아니라 `backlog-update`, `merge-doc-reconcile` 테스트/도구의 branch-specific 경로 계산까지 포함된다.
- 2026-05-01: 권장 fallback 정책은 `CODEX_WORKFLOW_BRANCH` 같은 명시적 환경변수를 최우선으로 두고, GitHub Actions 계열 환경변수(`GITHUB_HEAD_REF`, `GITHUB_REF_NAME`)를 보조로 사용한 뒤, 그래도 없으면 `main`으로 fallback하는 방식이다. 반환값은 파일시스템 경로로 쓰이므로 빈 문자열과 `HEAD`는 금지한다.
- 2026-05-01: WF-042-02 구현 완료. `get_current_branch()`가 `CODEX_WORKFLOW_BRANCH`, `GITHUB_HEAD_REF`, `GITHUB_REF_NAME`, `CI_COMMIT_REF_NAME`, `BRANCH_NAME` 순서로 안전한 branch slug를 찾고, `HEAD`/빈 값/절대경로/상위 경로 조각은 무시한 뒤 최종적으로 `main`으로 fallback한다.
