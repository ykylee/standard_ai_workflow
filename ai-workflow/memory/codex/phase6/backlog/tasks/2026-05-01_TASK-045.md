# TASK-045 Make workflow source development independent from applied runtime

- 문서 목적: `ai-workflow/` runtime/state 레이어가 없어도 `workflow-source/` 개발과 검증이 가능한 기준을 기록한다.
- 범위: runtime 의존 제거, 테스트 fixture 분리, 삭제 독립성 smoke test 추가
- 대상 독자: AI 에이전트, 개발자
- 최종 수정일: 2026-05-01
- 관련 문서: [../2026-05-01.md](../2026-05-01.md), [../../../../work_backlog.md](../../../../work_backlog.md), [../../session_handoff.md](../../session_handoff.md)

- 상태: done
- 우선순위: high
- 요청일: 2026-05-01
- 목표 버전: beta 0.4.0

## 배경

- 사용자가 `ai-workflow/` 폴더가 극단적으로 삭제되어도 workflow kit 개발에는 영향이 없어야 한다고 기준을 명확히 했다.
- `ai-workflow/`는 이 저장소에 적용된 runtime/state layer일 뿐이며, 개발 원본은 `workflow-source/`만으로 유지되어야 한다.

## 변경 사항

- `phase5_governance_guide.md`를 runtime memory에서 개발 원본인 `workflow-source/core/`로 이동했다.
- `export_harness_package.py`가 source docs export 시 `ai-workflow/memory/`가 아니라 `workflow-source/core/phase5_governance_guide.md`를 참조하도록 수정했다.
- `workflow_memory_dir()`가 `ai-workflow/memory/` 존재 여부와 무관하게 표준 runtime memory 경로를 계산하도록 수정했다.
- `check_workflow_state_generator.py`, `check_handoff_git_integration.py`가 현재 저장소의 `ai-workflow/memory/` 대신 `workflow-source/examples/acme_delivery_platform/` fixture를 사용하도록 바꿨다.
- `check_docs.py`가 삭제 독립성 확인 중 숨김 runtime 디렉터리와 선택적 `ai-workflow/` runtime 링크를 개발 문서 필수 링크로 취급하지 않도록 조정했다.
- `check_source_without_runtime_layer.py`를 추가해 `ai-workflow/`를 임시로 숨긴 상태에서 핵심 개발 검증을 수행하도록 했다.

## 검증

- `./.venv/bin/python workflow-source/tests/check_source_without_runtime_layer.py`
- `for t in workflow-source/tests/check_*.py; do ./.venv/bin/python "$t" || exit 1; done`
