## 정합 흐름 (memory → commit → push)

```
작업 결과 산출
  → memory 갱신 (state.json / session_handoff.md / work_backlog.md 등 active memory)
  → commit (memory 갱신을 *포함*)
  → push (협업자가 push 시점에 memory 변경까지 함께 봄)
```

## 안티패턴 (정정 대상)

- ❌ commit → push → 별도 turn memory 갱신 → 또 commit → push (memory 갱신 누락 / 추가 commit 유발)
- ❌ memory 갱신 누락 후 commit → push (협업자가 memory 변경을 push 시점에 못 봄)
- ❌ 별도 turn "memory 에 적어줘" 호출 (deprecated, 결과 commit 직전 같은 turn 에 묶어서 처리)

## 의도적 예외 (허용)

- `summarize_git_history` 로 commit hash 를 handoff 에 반영하는 작업은 commit 이후에 의미가 있다 — 다음 memory cycle 에 흡수하되 그 자체가 별도 commit 의 대상은 아니다.
