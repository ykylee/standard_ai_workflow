## Runtime layer 변경

### TASK-V11112-001: `cmd_validate` 5번째 source `mypy` 추가

**기존 4 source** (v0.7.9+):
1. `check_packaging.py` — pyproject ↔ 디스크 정합
2. `workflow_kit.cli.doctor` — 7 baseline evaluate
3. `state.json` freshness — last_freeze / last_ingest
4. `git status` — working tree clean

**5번째 source `mypy` 추가** (v0.11.12+):
- invocation: `python3 -m mypy --no-incremental <REPO_ROOT>/workflow_kit/`
  - cwd = `REPO_ROOT.parent` (project root)
  - target = absolute path (`REPO_ROOT/workflow_kit/`)
  - sub-package config merge 회피 (workflow_kit/pyproject.toml 의 strict=false 와 parent 의 strict=true merge 회피)
- result schema: `{ok, exit_code, error_count, first_error}`
  - mypy 부재 시: `{ok: False, error: "mypy module not installed (run pip install -e ./workflow-source/workflow_kit[dev])"}`
  - timeout 시: `{ok: False, error: "mypy timeout (>120s)"}`
- v0.11.11 의 CI invocation 과 동일 (= mypy 2.1.0 strict 0 errors 강제)

### TASK-V11112-002: `--skip-mypy` argparse flag

- 위치: `validate` subcommand 의 4 source skip flag 다음
- `action="store_true"` (default False = mypy check 활성)
- 다른 skip flag 와 일관 (`--skip-packaging` / `--skip-doctor` / `--skip-state` / `--skip-git`)
- `cmd_release` 의 `getattr(args, "skip_mypy", False)` 로 정합

### TASK-V11112-003: `release_pipeline_lib.cmd_release` 3 kwarg forwarding

**기존 signature**:
```python
def cmd_release(*, version: str, notes_template: str | None = None,
                skip_validate: bool = False, auto_bump: bool = False,
                apply: bool = False) -> dict:
```

**변경 후**:
```python
def cmd_release(*, version: str, notes_template: str | None = None,
                skip_validate: bool = False, skip_mypy: bool = False,
                auto_bump: bool = False, full_auto: bool = False,
                allow_existing_tag: bool = False,
                apply: bool = False) -> dict:
```

3 kwarg 추가 + `_make_args(...)` 에 forwarding:
- `skip_mypy` — v0.11.12+ (본 release 의 핵심)
- `full_auto` — v0.9.1+ (기존 `cmd_release` 가 `getattr(args, "full_auto", False)` 사용하지만 dispatcher 미 forwarding = no-op 버그 fix)
- `allow_existing_tag` — v0.7.21+ (기존 `cmd_release` 가 `getattr(args, "allow_existing_tag", False)` 사용하지만 dispatcher 미 forwarding = no-op 버그 fix)

**In-scope fix** (release pipeline consistency): `full_auto` / `allow_existing_tag` 의 dispatcher forwarding 은 본 release 의 mypy pre-check 과 직접 관련 (둘 다 `cmd_release` 의 race condition fix). v0.9.1~v0.11.11 사이의 9 release 가 이 flag 들을 dispatcher 통해 호출 불가능했음.

### TASK-V11112-004: `_make_args` default fill

**기존**: `SimpleNamespace(**kwargs)` — kwargs 에 없는 attribute 는 AttributeError
**변경 후**: validate 관련 5 source skip flag (skip_packaging / skip_doctor / skip_state / skip_git / skip_mypy) + skip_validate 를 default False 로 자동 fill

**근거**: v0.11.11 까지 dispatcher 가 `cmd_release` 호출 시 4 source skip flag 를 전달하지 않아 `args.skip_packaging` AttributeError. 본 release 에서 5 source skip flag 모두 default fill 로 fix.
