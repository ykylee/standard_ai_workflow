### TASK-V11113-001: `_cross_verify_ci_mypy` helper

**Invocation**: `gh run list --repo ykylee/standard_ai_workflow --workflow mypy-strict.yml --limit 1 --json databaseId,conclusion,headSha,event,status,createdAt,url`

**CI-only verdict кІ°м •** (Layer 1 кІ°кіј):
| ci_conclusion | headSha match | CI-only verdict |
|---|---|---|
| success | yes | `ci_sanity` |
| success | no | `ci_stale` |
| failure | (any) | `ci_fail` |
| other / absent / no run | (any) | `absent` |
| gh CLI л¶Җмһ¬ / error / timeout | вҖ” | `skipped` |

**Returns**:
```python
{
    "verdict": "ci_sanity" | "ci_stale" | "ci_fail" | "absent" | "skipped",
    "ci_run": {databaseId, conclusion, headSha, event, status, createdAt, url} | None,
    "head_sha": str | None,
    "head_sha_match": bool | None,
    "message": str,
}
```

### TASK-V11113-002: `_resolve_cross_verify_verdict` helper

CI-only verdict + local mypy вҶ’ final verdict (Layer 1 вҶ” Layer 2 м •н•©):

| CI verdict   | local mypy ok | local status | final verdict      |
|--------------|---------------|--------------|--------------------|
| ci_sanity    | True          | checked      | **sanity**         |
| ci_sanity    | False         | checked      | **drift_warning**  |
| ci_sanity    | N/A           | skipped      | **no_local_verify** |

| ci_fail      | (any)         | (any)        | ci_fail            |
| absent       | (any)         | (any)        | absent             |

### TASK-V11113-003: `cmd_release` нҶөн•©

```
1. cross-verify (_cross_verify_ci_mypy) вҖ” Layer 1 CI result
2. validate (cmd_validate) вҖ” 5 source Layer 2 gate
2.5. verdict кІ°н•© (_resolve_cross_verify_verdict) вҖ” local mypy мҷҖ cross
```

**--strict-cross-verify flag мӢң hard fail**:
- `drift_warning`: CI success мқёлҚ° local mypy fail (Layer 1 вҶ” Layer 2 drift)
- `ci_stale`: CI success мқёлҚ° headSha != HEAD (re-run к¶Ңкі )
- `ci_fail`: CI fail (regardless of local)
- (`no_local_verify` / `absent` / `skipped` лҠ” strict лҸ„ advisory)
