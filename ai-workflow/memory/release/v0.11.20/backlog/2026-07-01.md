## Lesson

### cache.py state.json path latent bug

v0.6.0.1 의 `memory/active/` rename + PROJECT_PROFILE.md 동시 migration 시점에 cache.py 의 `/ "active"` 추가가 PROJECT_PROFILE.md 의 새 위치 (`<memory>/active/`) 와 정합되지 않았음. 그 후 모든 skill 의 `refresh_workflow_state_cache` 호출이 dead path 에 state.json 을 쓰고 있었으나 production 에서는 `--output-path` 를 명시적으로 지정해 우회. smoke test 의 `check_code_index_update_apply.py` 가 silently fail 했으나 v0.11.18 의 FULL mypy strict 와 별개로 발견.

**lesson (cross-project)**: directory rename 시 모든 path-computing helper 가 *두 곳* (production layout + test fixture layout) 양쪽과 정합되는지 verify. `apply` test 가 silent fail 한 경우 명시적 exit 1 로 fail-fast 변환.

### workflow_linter `.absolute()` + `..` 정규화 함정

v0.7.22 의 `.resolve()` → `.absolute()` fix (symlink-aware) 가 *의도적* 으로 symlink 따라가지 않음은 좋았지만 `..` segment 정규화도 함께 fallback. 결과적으로 `/tmp/x/foo/../../bar.md` 형태의 literal path 가 `.exists()` 에서 False.

**lesson (cross-project)**: symlink 보존이 필요한 helper 와 path 정규화가 필요한 helper 는 분리. 또는 `os.path.normpath` 후 `os.path.realpath` (symlink resolve) 의 조합으로 양립.

### project-status-assessment Pydantic schema 정합

`build_runner_success_result` + `extra_fields` dict emission 이 다른 stable skill 의 `BaseOutput` + nested Pydantic 패턴과 비대칭. 정합 후:
- `validate_output_payload(family="project_status_assessment")` 자동 검증 가능
- 다른 stable skill consumer 가 동일 interface 로 통합 가능
- 신규 skill 추가 시 schema 등록 절차 표준화

**lesson (cross-project)**: stable skill 의 output schema 는 처음부터 `BaseOutput` + nested dataclass + `output_contracts.PYDANTIC_MODEL_REGISTRY` 등록 패턴 사용. legacy dict emission ❌.
