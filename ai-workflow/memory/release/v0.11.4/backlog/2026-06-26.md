## Runtime layer 변경

### TASK-V1114-001: output_contracts.py 6 errors → 0

**Error 분석** (mypy 2.1.0):
```
output_contracts.py:296 error: Incompatible types in assignment
  (expression has type "type[BaseModel] | None", variable has type "type[ErrorOutput]")
output_contracts.py:297 error: Function "model_cls" could always be true in boolean context
output_contracts.py:323 error: Dict entry 0 has incompatible type
  "str": "dict[str, dict[str, object]]"; expected "str": "dict[str, object]"
output_contracts.py:324 error: Dict entry 1 has incompatible type ... (동일 패턴)
output_contracts.py:357 error: "object" has no attribute "get"
output_contracts.py:368 error: Incompatible return value type
  (got "object", expected "dict[str, object]")
```

**Fix 정공법**:
1. **line 296-297 (assignment + truthy-function)**: `model_cls: type[BaseModel] | None = ...; if model_cls is None: return ...` 패턴. 단 `model_cls = ErrorOutput` (line 293) 의 경우 `type[ErrorOutput]` 으로 narrow 필요. fix: `if not model_cls` 후 `assert model_cls is not None` 또는 `cast(type[BaseModel], model_cls)`.
2. **line 323-324 (dict-item)**: `errors: dict[str, dict[str, object]] = {...}` 또는 return type 명시. fix: `errors = {family: ... for family in ...}` 패턴 + 명시적 `dict[str, dict[str, object]]` annotation.
3. **line 357 (attr-defined)**: `object.get(...)` → `cast(dict, model_cls).model_json_schema()` 또는 명시적 `getattr(obj, "get", lambda *_: {})(...)`.
4. **line 368 (return-value)**: `dict[str, object]` 명시 return 시 `dict` literal annotation.

**Fix 정공법**:
