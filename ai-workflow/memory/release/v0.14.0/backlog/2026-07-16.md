## 2. 메모리 layout (v0.14.0+ 정공법)

```
ai-workflow/memory/active/
├── state.json                     ← read-only snapshot (state/builder.py 가 rebuild)
├── state.json.template            ← unchanged
├── PROJECT_PROFILE.md / PURPOSE.md / project_status_assessment.md / repository_assessment.md / session_analysis_<date>.md
├── backlog/                       ← 🆕 per-day + per-task append-only
│   ├── YYYY-MM-DD.md              ← per-day index (append-only)
│   └── tasks/
│       ├── TASK-YYYY-MM-DD-001.md ← 1 file = 1 task (per-task SSOT, frontmatter 정합)
│       └── ...
├── sessions/                      ← 🆕 per-session (raw_path stem 그대로)
├── memory_index/                  ← unchanged (이미 append-only 정공법)
└── log.md                         ← append-only
```

**충돌 표면 (multi-agent 동시 작업 시)**:
- `state.json`: `--apply` flag 로 1 agent 만 rebuild, atomic_write 보장
- `backlog/<date>.md`: 동일 날 다른 agent 가 다른 task 추가 시 3-way merge (양쪽 concat 으로 해결 가능)
- `backlog/tasks/TASK-XXX.md` / `sessions/<sid>.md` / `memory_index/entries/*.json`: **물리 격리** (서로 다른 파일)
- `memory_index/telemetry/events.jsonl` / `log.md`: **append-only** (이미 적립)

→ **mutable 공유 파일이 `state.json` 단 1개**. 나머지는 모두 append-only 또는 자기 소유.
