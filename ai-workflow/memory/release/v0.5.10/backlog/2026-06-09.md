### 작업 내용

저장소 사용자/통합 담당자가 자주 묻는 "소스에서 어떻게 설치하고 어떻게 쓰는가"에 대한 **단일 권위 가이드**를 `docs/` (영구 지식 문서) 에 추가한다.

배경:
- 기존 `README.md` 와 `QUICKSTART.md` 는 **소비자 관점** (미리 빌드된 zip 패키지를 받아 다른 프로젝트에 적용) 만 다룬다.
- 개발자가 `git clone` 후 소스에서 editable install 하고 스모크 테스트를 돌리고 `workflow_kit` / `bootstrap_lib` 를 임포트해 쓰는 절차는 어디에도 한 자리에 정리돼 있지 않다.
- `pyproject.toml` 이 `workflow-source/` 안에 있고, editable install 시 cwd 가 `workflow-source/` 여야 하는 함정, `PYTHONPATH=workflow-source` 가 필요한 경로, `.[mcp-sdk,dev]` extras 의미 등이 흩어져 있다.

  - 환경 진단 한 줄 스모크
  - 52개 스모크 테스트 일괄 실행 / 개별 실행 / 실패 트리아지
