### 작업 내용

오케스트레이터가 작업을 직접 도구 호출로 처리하지 않고 `mini-coder-max` 또는 `general` 워커에게 위임하는 contract v1 설계/문서화/검증.

contract v1 의 1차 컷 스코프:
- **입력 스키마 (orchestrator → sub-agent)**: `{task_id, task_type, brief, constraints, handoff_path, validation_required}` JSON
- **출력 스키마 (sub-agent → orchestrator)**: `{status, summary, artifacts, written_paths, validation_result, next_step}` JSON
- **4개 역할 경계**: orchestrator / doc-worker / code-worker / validation-worker
- **위임 가능 작업 카탈로그**: 대량 파일 read, 문서 비교, bounded patch, 빌드/테스트 실행, 로그 수집
- **위임 불가 / 직접 처리 작업**: 짧은 triage read, handoff 갱신, ask_user 호출
- **에러 처리 정책**: 워커 실패 시 orchestrator 가 retry (1회) → 직접 처리 (폴백) → 사용자에 상향
- **검증 시나리오 1개**: TASK-V054-001 의 contract doc 작성 자체를 "orchestrator 가 doc-worker 에게 위임" 으로 실행하고, 그 결과를 PR 본문에 명시
