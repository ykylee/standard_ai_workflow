## In-scope fix 동반

- **fix 1 (real, defensive cast)**: `cast(list[str], handoff.get("constraints") or [])` — None 방어. 상위 caller 가 `{"constraints": None}` 명시 시 `cast` 가 type-lying → `or []` 추가가 정공법. *현장 fixture* 에서는 [] default 이므로 PASS 했지만, defensive programming 측면에서 in-scope fix 동봉.
- **fix 2 (real, regex literal)**: `re.match(r"^---\n(.+?)\n---", text, re.S)` — frontmatter 가 file *시작* 이 아닌 경우 match fail. *현재는 v0.9.2 PURPOSE.md 가 file 시작이므로 PASS* 이지만, helper robustness 측면에서 *의도적 re.match 유지*. 후속 release 에서 file 시작 검증 추가 검토 (R-A follow-up cycle 외 scope).
