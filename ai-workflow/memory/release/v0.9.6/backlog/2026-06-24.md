## Acceptance criteria (spec §4.4 7 detail)

1. `parse_log_events` 가 `## [YYYY-MM-DD] <event> | <summary>` 형식의 wiki log event 들을 정확히 parse
2. `analyze_30day_distribution` 가 30일 window 안 event 들을 ingest / query / release 분류 + top 10 topics + recent releases
3. `update_last_purpose_review` 가 PURPOSE.md frontmatter 의 `last_purpose_review` date 를 `re.MULTILINE` regex 로 정확히 갱신 + 다른 field 보존
4. `generate_llm_suggest_prompt` 가 markdown 형식의 §1 분포 + §2 본문 ≤800 char + §3 4-element advisory 요청 emit + auto-commit ❌ 명시
5. `run_purpose_refresh(apply=False)` 일 때 file I/O ❌ (dry-run) + prompt 정상 emit
6. `run_purpose_refresh(apply=True)` 일 때 frontmatter `last_purpose_review` date 갱신 + `updated=True`
7. log.md / PURPOSE.md 부재 시 graceful skip + advisory warning 1줄 + no-op (auto-fail ❌)
8. (CLI smoke) `workflow_kit_cli --command=refresh-purpose` dry-run + `--apply` 2 case 정상 동작
