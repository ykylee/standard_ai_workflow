### 3.2 ensure_stage_completion() lazy fallback

```python
def ensure_stage_completion(result: dict) -> dict:
    """result dict 에 stage_completion 이 없으면 *자동 생성*."""
    if "stage_completion" not in result:
        # legacy code / CI / cron / PoC 호환
        result["stage_completion"] = {
            "stage_name": "auto_generated",
            "stage_status": "done",
            "approval_timestamp": now_iso8601(),
            "approval_actor": "auto",
            "artifacts": [],
            "notes": ["auto-generated by ensure_stage_completion"],
        }
    return result
```

**호환 케이스**:
- Legacy code: stage_completion 없이도 통과 (auto-generate)
- CI/cron: 동일
- PoC: 동일
- Production: 명시적 stage_completion required (ensure_stage_completion 의 auto-generate 없음)

## §4 Auto-approval 한계  {#s4-auto-approval}

`require_explicit_approval()` 의 `is_production_code` / `is_state_doc` / `is_release` 가 True 일 때 auto-approval 거부.

```python
def require_explicit_approval(
    completion: StageCompletion,
    is_production_code: bool = False,
    is_state_doc: bool = False,
    is_release: bool = False,
) -> bool:
    """production / state / release 변경 시 명시적 approval 강제."""
    if is_production_code or is_state_doc or is_release:
        if completion.approval_actor == ApprovalActor.AUTO:
            raise GateValidationError(
                "auto-approval rejected: production/state/release requires explicit user approval"
            )
    return True
```

**Notes 에 auto-approval 사유 keyword 포함**: `p0 hotfix` / `ci timeout` / `cron` / `auto-approved` 등.

## §7 한계 / 예외  {#s7-limitations}

- **PoC / Legacy code**: ensure_stage_completion 의 auto-generate 가 stage_completion *quality* 저하 가능
- **CI / cron**: 동일. *명시적 stage* 권장 (CI job name + commit hash 포함)
- **state doc (audit.md, state.json, release.md)**: production-grade 동급 — explicit approval 강제
