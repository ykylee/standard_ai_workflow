# Standard AI Workflow

The repo `standard_ai_workflow` (Standard AI Workflow). **Hub entity** for the L1 runtime wiki.

## Role

독립 패키지 형태로 관리되는 **cross-project AI 협업 워크플로우 표준 + 구현 키트**. 문서와 템플릿만으로 최소 운영이 가능하며, 같은 구조 안에서 `skill`, `MCP`, `agent` 구현이 점진 확장된다.

대상 독자: 개발자, 운영자, AI agent 설계자, 프로젝트 온보딩 담당자.

**핵심 원칙 (SSOT):**

- 공통 규칙은 `workflow-source/core/`에 둔다. 저장소별 차이는 `project_workflow_profile_template.md`에.
- `ai-workflow/memory/active/*`는 **workflow state docs** (session 복원, handoff, state cache); 실제 프로젝트 운영 문서(`docs/...`)와 역할 분리.
- skill/MCP/agent는 설계 카탈로그 우선, 실제 구현은 프로젝트 상황에 맞게 선택 적용.
- 외부 의존 링크 최소화 — 이 저장소만 읽어도 구조 파악 가능.

**대상 하네스 (6종, 모두 first-class):**

| Harness | Entry Point | Overlay Pattern |
|---|---|---|
| Mavis (MiniMax Code) | `MiniMax.md` + `AGENTS.md` + `.minimax/agents/` | orchestrator + doc/code/validation worker |
| Codex CLI | `AGENTS.md` + `.codex/config.toml.example` | task-only orchestrator + bounded worker |
| OpenCode CLI | `AGENTS.md` + `opencode.json` + `.opencode/...` | orchestrator + generic/specialized worker |
| Gemini CLI | `GEMINI.md` | entry-point overlay |
| Antigravity | `ANTIGRAVITY.md` | entry-point overlay |
| pi-dev | `AGENTS.md` + `SYSTEM.md` | agent persona overlay |

신규 하네스 추가는 `HARNESS_SPECS` 한 줄 + `register_harness_builder` 한 줄로 끝난다.

## Composition

| Directory | Type | Version Sync | Key Role |
|---|---|---|---|
| `workflow-source/` | SSOT (git, PR-reviewed) | tracks repo | 코어 표준 문서, 템플릿, skill/MCP 프로토타입, Python package, 테스트, 릴리스 노트 |
| `ai-workflow/` | Runtime (mixed, `.gitignore`) | regenerated by bootstrap | session state, handoff, state cache, harness entry, MCP config, scripts |
| `workflow_kit/` | Python package (git, wheel) | tracks repo | `workflow_kit`, `bootstrap_lib`, `contract_v1/`, `common.{state,contracts,schemas,server,...}` (30+ submodules) |
| `docs/` | Project docs (git) | tracks repo | `INSTALLATION_AND_USAGE.md`, `RELEASE.md`, `PROJECT_PROFILE.md` |
| `dist/` | Harness artifacts (git) | regenerated by `export_harness_package.py` | `dist/harnesses/<target>/<version>/` + zip; `PACKAGE_CONTENTS.md`, `APPLY_GUIDE.md` |

**3-Layer 분리** ([[decisions/adr-001-3-layer-separation]]):

1. **Source** (`workflow-source/`) — git, PR-reviewed, canonical.
2. **Runtime** (`ai-workflow/`) — generated per project, mixed VCS.
3. **Distribution** (`dist/`) — packaged harness bundles for offline adoption.

내부 모듈: 11종 skill 프로토타입 (1차 6 + 2차 2 + 3차 3) + 우선순위 1 MCP 실행형 초안 + `read_only_mcp_sdk v1.0` candidate. `workflow-source/tests/check_*.py` **52개** smoke test가 CI에서 매 push마다 전부 실행 (python 3.11, `PYTHONPATH=workflow-source`).

## Adoption Patterns

### A. New project (template-driven)

1. `workflow-source/core/global_workflow_standard.md`를 기준 문서로 읽는다.
2. `templates/project_workflow_profile_template.md`를 root에 복사 후 프로젝트 특화 규칙을 채운다.
3. `templates/`의 session/backlog 템플릿을 배치한다.
4. `core/workflow_skill_catalog.md`, `core/workflow_mcp_candidate_catalog.md`, `core/workflow_agent_topology.md`로 도입 범위를 정한다.
5. 영향이 큰 흐름 (session 시작, backlog 갱신, doc-sync)부터 시작한다.

```bash
python3 -m bootstrap_lib \
  --target-root /tmp/sample-repo \
  --project-slug sample_api --project-name "Sample API" \
  --harness codex --harness opencode --harness antigravity \
  --copy-core-docs
```

### B. Existing project (analysis-driven)

```bash
python3 -m bootstrap_lib \
  --target-root /tmp/existing-repo \
  --project-slug payments_api --project-name "Payments API" \
  --adoption-mode existing --harness codex --copy-core-docs
```

`adoption-mode existing`은 `repository_assessment.md` 자동 생성을 trigger; 추정값을 실제 repo 규칙과 대조해 수정해야 한다.

### C. 6-Harness overlay

[[concepts/harness-distribution]] 참조. 주요 overlay:

- **Mavis (MiniMax Code)** — 권장, orchestrator/worker 자동 emit, `.minimax/agents/` 구조 생성
- **Codex** — `AGENTS.md` + `.codex/config.toml.example` (또는 `.codex/mcp.toml`)
- **OpenCode** — `AGENTS.md` + `opencode.json` + `.opencode/...` (project-local skill/agent)
- **Gemini CLI** — `GEMINI.md`
- **Antigravity** — `ANTIGRAVITY.md`
- **pi-dev** — `AGENTS.md` + `SYSTEM.md` (persona)

`--enable-mcp`로 read-only MCP config 스니펫 동시 emit. transport:
- `--mcp-bridge jsonrpc-bridge` (default, **안정**)
- `--mcp-bridge stdio-sdk` (실험적, known connection-closed regression)

### D. Self-dogfooding (이 저장소 자체 개발 시)

```bash
# 1) 런타임 동기화
python3 -m bootstrap_lib --target-root . --project-slug standard-ai-workflow \
  --project-name "Standard AI Workflow" --harness antigravity \
  --adoption-mode existing --copy-core-docs --force

# 2) state.json 갱신
python3 workflow-source/scripts/generate_workflow_state.py \
  --project-profile-path docs/PROJECT_PROFILE.md \
  --session-handoff-path ai-workflow/memory/active/session_handoff.md \
  --work-backlog-index-path ai-workflow/memory/active/work_backlog.md \
  --output-path ai-workflow/memory/active/state.json
```

> 첫 체크아웃 후 bootstrap을 반드시 실행해 `ai-workflow/`와 harness entry 파일을 로컬에 생성할 것 (둘 다 `.gitignore` 대상).

### E. Export harness package (offline 배포)

```bash
python3 workflow-source/scripts/export_harness_package.py --harness codex --harness opencode
```

`dist/harnesses/<target>/<version>/` 아래 zip. 기본은 runtime만; source docs / global snippets는 opt-in.
