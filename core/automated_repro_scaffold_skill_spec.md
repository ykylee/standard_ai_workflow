# Skill Spec: automated-repro-scaffold

- 문서 목적: 버그 재현 환경 자동 구축 스킬의 입력, 출력, 판단 로직을 정의한다.
- 범위: 버그 리포트 분석, 재현용 샌드박스 스크립트 생성
- 대상 독자: AI 에이전트 개발자, 워크플로우 설계자
- 상태: draft
- 최종 수정일: 2026-04-27
- 관련 문서: `skills/automated-repro-scaffold/SKILL.md`, `core/workflow_kit_roadmap.md`

## 1. 역할
사용자가 보고한 버그 내용(또는 테스트 실패 로그)을 분석하여, 해당 현상을 독립적으로 재현할 수 있는 최소 단위의 파이썬 스크립트(`repro_*.py`)를 자동으로 생성한다.

## 2. 입력 (Input Contract)
- `bug_report`: 버그 설명, 기대 동작, 실제 동작, 재현 단계 등을 포함한 텍스트.
- `existing_test_code` (선택): 참고할 수 있는 기존 테스트 파일 내용.
- `project_profile`: 프로젝트의 검증 명령 및 환경 제약 사항.

## 3. 출력 (Output Contract)
- `repro_script_path`: 생성된 재현 스크립트의 경로.
- `repro_script_content`: 스크립트의 전체 내용.
- `execution_command`: 스크립트를 실행하기 위한 명령 (예: `python3 repro_bug.py`).
- `warnings`: 의존성 라이브러리 부재 등 실행 시 주의사항.

## 4. 판단 로직 (Decision Logic)
1. **버그 리포트 분석**: 리포트에서 에러 메시지, 호출 스택, 특정 입력값을 추출한다.
2. **샌드박스 설계**: 프로젝트 외부 의존성을 최소화한 상태에서 해당 로직을 호출하는 코드를 구상한다.
3. **템플릿 적용**: `unittest` 또는 단순 `assert` 문을 포함한 독립 실행형 스크립트 구조에 내용을 채운다.
4. **검증 안내**: 생성된 스크립트가 실패(버그 재현 성공)해야 함을 명시한다.

## 5. 실패 시나리오
- 버그 리포트 내용이 너무 모호하여 재현 로직을 특정할 수 없는 경우.
- 특정 하드웨어나 외부 API 의존성이 강해 로컬 샌드박스 생성이 불가능한 경우.
